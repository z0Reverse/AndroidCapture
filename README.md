## 一、项目简介
`redirect.sh` 是 Android Root 设备专用的 iptables 四层透明流量转发脚本，解决 Wi-Fi 系统代理抓包卡顿、App 绕过代理等问题。
核心能力：
1. 全局转发整机所有 TCP 流量至抓包工具 8080 透明代理端口
2. 定向转发单个 App 流量（支持纯数字UID / `u0_aXXX` 安卓用户名自动换算）
3. 一键清空 NAT 转发规则、查看当前已存在转发规则
4. 内置参数校验、防重复添加规则、ROOT权限检测、错误提示

适用场景：安卓 App 抓包、接口调试、流量劫持、代理转发，相比系统 Wi-Fi 代理延迟更低、不丢包、性能接近原生网络。

## 二、环境要求
1. Android 设备已获取 Root 权限（必须，修改 iptables 需要内核权限）
2. 设备内置 `iptables`、`/system/bin/sh`（原厂安卓均自带）
3. 抓包工具（BurpSuite/Charles/Fiddler）开启**透明代理**，监听 `8080` 端口
4. 手机与电脑处于同一局域网（互通 IP）

## 三、安装部署
1. 将脚本上传至安卓设备（示例路径 `/data/local/tmp/redirect.sh`）
2. 赋予可执行权限
```shell
su
chmod 755 /data/local/tmp/redirect.sh
```
3. 执行脚本查看帮助文档，验证可用
```shell
su -c /data/local/tmp/redirect.sh
```

## 四、完整使用说明
### 1. 命令格式
```shell
./redirect.sh [选项] [UID/u0_a用户名] [目标IP]
```

### 2. 全部参数说明
| 参数 | 功能 |
|------|------|
| `-S` | 查看当前 NAT 表所有转发规则，用于排查重复规则、流量不转发问题 |
| `-F` | 一键清空 nat 表全部转发规则，抓包结束必执行，避免残留转发 |
| `-A 目标IP` | 全局转发：设备所有应用 TCP 流量转发至 `IP:8080` |
| `UID/u0_aXXX IP` | 定向转发：仅转发指定 App TCP 流量，支持两种UID格式 |

### 3. UID 两种输入格式（自动兼容）
1. 纯数字UID：`10156`
2. 安卓应用用户名：`u0_a156`（脚本内部自动换算为数字UID 10156）

换算规则：
- 主用户 `u0_aXX`：`0 * 100000 + 10000 + XX`
- 多用户工作空间 `u1_aXX`：`1 * 100000 + 10000 + XX`

### 4. 实操示例
```shell
# 1. 查看帮助文档
su -c /data/local/tmp/redirect.sh

# 2. 查看当前所有转发规则
su -c /data/local/tmp/redirect.sh -S

# 3. 全局转发所有App流量到电脑 192.168.1.100:8080
su -c /data/local/tmp/redirect.sh -A 192.168.1.100

# 4. 定向转发，使用u0_a用户名（推荐，直接复制ps输出）
su -c /data/local/tmp/redirect.sh u0_a156 192.168.1.100

# 5. 定向转发，使用纯数字UID
su -c /data/local/tmp/redirect.sh 10156 192.168.1.100

# 6. 抓包完成，清空所有转发规则（重要！）
su -c /data/local/tmp/redirect.sh -F
```

## 五、脚本内置安全校验机制
1. ROOT 检测：非 root 用户直接报错退出，禁止执行
2. IPv4 合法性校验：拒绝非法 IP、网段数值超出 0-255 的地址
3. UID 格式校验：仅允许纯数字 / `uX_aXXX` 格式，非法输入直接报错
4. 重复规则检测：相同转发规则不会重复叠加，防止流量翻倍、网络卡顿
5. 命令执行捕获：iptables 操作失败会输出明确错误信息并退出

## 六、抓包卡顿优化配套方案
1. 电脑抓包工具开启**透明代理模式**，监听 8080 端口
2. 关闭工具实时日志渲染、自动保存完整响应体、断点调试，降低 CPU 占用
3. 手机连接 5GHz Wi-Fi，减少无线干扰丢包
4. 抓包结束务必执行 `-F` 清空规则，防止长期转发占用网络
5. 优先使用定向UID转发，仅劫持目标App，整机网络不受影响

## 七、常见问题
### Q1：执行提示权限不足
A：脚本必须加 `su -c` 执行，或先 `su` 进入root环境再运行。

### Q2：流量没有转发到抓包工具
1. 执行 `./redirect.sh -S` 检查规则是否成功添加
2. 确认手机与电脑局域网互通，关闭电脑防火墙/杀毒软件拦截8080端口
3. 抓包工具必须开启透明代理，普通HTTP代理无法接收DNAT转发流量

### Q3：u0_aXXX 和数字UID两条命令效果一样吗？
A：UID转换后完全一致，仅目标IP不同则转发地址不同；IP相同则两条命令等价。

### Q4：重启手机后转发规则失效
A：iptables NAT 规则为临时内存规则，重启系统自动清空，如需持续抓包需重新执行脚本。

## 八、脚本退出码规范（便于自动化调用）
| 退出码 | 含义 |
|--------|------|
| 0 | 执行成功 |
| 1 | 非ROOT权限 |
| 2 | IP地址格式非法 |
| 3 | UID输入格式错误 |
| 4 | 清空nat规则失败 |
| 5 | 参数数量/选项非法 |
| 6 | 全局转发规则添加失败 |
| 7 | UID定向转发规则添加失败 |
| 55 | 原始脚本预留参数错误码 |
